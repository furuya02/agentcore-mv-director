# MV生成ルール（うまくいく構成のまとめ）

開発の試行錯誤で確立した「破綻しにくいMV生成の作法」。今後の実装・拡張の基準とする。

## 基本ルール

1. **8秒を基本単位にする**
   - PixVerse の i2v / extend はいずれも 8秒単位。Kling LipSync の入力上限は10秒なので、8秒なら安全に収まる。
2. **各シーンは「与えられた画像」から始める（image-to-video）**
   - 画像の品質がそのまま活き、アスペクト比の歪みや連鎖劣化を避けられる。
3. **シーン内の連続は PixVerse の extend で繋ぐ**
   - extend は「続き」を生成するので、フレーム連鎖より連続性が良い。
4. **リップシンクは「8秒単位で切り出した動画」に当てる**
   - 連続動画を8秒チャンクに分割し、各チャンクを **その時間帯の歌声** で同期 → 連結。
   - これで「連続性」と「全編リップシンク」を両立できる。

## 用語

- **シーン**：1枚の画像から作るクリップ。
  - 複数画像モード（`--images`）：1画像 = **8秒**（i2v 1本）。総尺 = 8 × 画像枚数。
  - 単一画像の連続モード（`--image --extend N`）：8 ×(1 + N) 秒の**連続クリップ**（extend で延長）。
- **チャンク**：リップシンクのための8秒スライス。

## 例

- **画像3枚（`--images`）** → 各8秒 → **24秒**のMV（5枚なら40秒）。Claude vision がカメラ目線のアップ画像を自動判定してリップシンク適用。
- **画像1枚（`--images`）** → 既定 **24秒**（同じ画像を8秒×3に複製。`--length` で変更可）。
- **1枚を連続にしたい（`--image X --extend 2`）** → 8×3=**24秒の連続動画**、8秒チャンクごとに全編リップシンク。

## 重要な注意（ハマりどころ）

- **extend は累積を返す**：PixVerse extend の出力は「元動画＋延長分」。シーンのクリップとして使うのは **最後の extend 出力だけ**（中間の extend を連結すると重複する）。
- **リップシンク対象は Claude vision が自動判定**：複数画像モードでは、各画像を Bedrock（Claude）で分析し、人物がカメラ目線でアップに映っている場合のみリップシンク（歌唱カット）に設定する。ファイル名の命名規則は不要。後ろ向き・風景などは自動的に映像のみになる。並び順はファイル名中の数字（連番）で決まる。リップシンク対象画像は**正面の顔が必須**（内部で顔検出するため。顔が無いと `face_detection_error`）。
- **音声は時間帯を合わせる**：チャンク i には楽曲の `[i×8, (i+1)×8]` 秒を渡す（先頭8秒を全チャンクに渡すと後半の口がズレる）。
- **楽曲の長さ = 動画の総尺**：`length_ms = 総秒数 × 1000`。
- **解像度・アスペクト**：i2v は入力画像のアスペクト比を踏襲。狙いの比率に**事前クロップ**。連結前に解像度・fpsを揃える（scale + fps 正規化）。
- **チャンク境界**：チャンクを独立にリップシンクするため、8秒・16秒…の境目で顔が軽く"飛ぶ"ことがある。気になる場合はクロスフェード等で緩和。
- **時間・コスト**：リップシンクは1チャンク約12分。**顔チャンク数 × 12分**かかる（例：6チャンク全部顔なら約72分）。`REUSE=1` で生成済みを再利用し、失敗時は途中再開する。

## 処理フロー（1シーン分）

```
画像 →[prepare_image: 3:2クロップ]→ [PixVerse i2v 8秒]
     →[PixVerse extend ×N]→ 連続クリップ(8×(1+N)秒, 累積の最後を採用)
     →[8秒ごとに cut_segment]→ チャンク群
       └ 各チャンク（顔あり）→[Kling LipSync: 該当時間帯の歌声]→ リップシンク済みチャンク
     →[FFmpeg 連結＋楽曲合成]→ シーン動画
複数シーンは最後に連結して1本のMVへ。
```

## コマンド対応

- 複数画像（各画像＝1シーン、extend で延長、顔ありは全編リップシンク）：
  - `python -m scripts.run "コンセプト" --images input --extend 1`
  - `--extend 0` なら各シーン8秒、`--extend 1` なら各シーン16秒。
- 単一画像から連続生成：`python -m scripts.run "コンセプト" --image input/001.png --extend 2`
- 出力先を分ける：先頭に `OUTPUT_DIR=フォルダ名` を付ける。
- 途中再開／再利用：先頭に `REUSE=1`。
